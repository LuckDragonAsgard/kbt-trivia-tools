### Conversion mechanics

- Default `handler(req, res)` → `export const onRequestPost = async ({ request, env })`
- `process.env.X` → `env.X`
- `req.body` → `await request.json()`
- `req.query.x` → `new URL(request.url).searchParams.get('x')`
- `res.status(s).json(d)` → `Response.json(d, {status: s, headers: CORS})`
- `Buffer.from(b64, 'base64')` → `Uint8Array.from(atob(b64), c => c.charCodeAt(0))`

`generate-slides.js` already used Web Crypto API (`crypto.subtle.importKey`, `crypto.subtle.sign`) and `btoa`/`atob` — no conversion needed for the JWT signing logic. Just the wrapper.

## Paddy's one-time deploy step

Cloudflare doesn't auto-detect new repos — you have to wire it once via dashboard:

1. dash.cloudflare.com → **Workers & Pages** → **Create** → **Pages** → **Connect to Git**
2. Choose `LuckDragonAsgard/kbt-trivia-tools` → keep defaults (no build command, output dir = `/`)
3. **Settings → Environment variables** (set for both Production AND Preview):
   - `FAL_KEY` — fal.ai API key
   - `ANTHROPIC_API_KEY` — Anthropic API key
   - `GOOGLE_SA_JSON` — Google Service Account JSON for slide generation
4. Trigger redeploy (or just push anything to main)
5. Test `https://<project>.pages.dev/api/env-check` — should return all keys YES

Once green, optionally point a custom domain (e.g. `tools.knowbrainertrivia.com.au`) at the pages.dev URL.
