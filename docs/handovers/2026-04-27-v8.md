## How the next session should resume the migration

The push pipeline is: **download from Drive (base64) → PUT to GitHub Contents API**. No CodeMirror, no zip uploads. Tested through 13 markdown commits this session.

### Step 1 — get the GH token

Pull `GITHUB_TOKEN` from `https://asgard-vault.pgallivan.workers.dev` (X-Pin: 2967). Scope needs to be `public_repo` (or full `repo`).

### Step 2 — write the helper

```python
# /tmp/kbt_push.py
import sys, json, urllib.request, urllib.error
target, msg, b64 = sys.argv[1], sys.argv[2], sys.stdin.read().strip()
sha = None
url = f'https://api.github.com/repos/LuckDragonAsgard/kbt-trivia-tools/contents/{target}'
hdr = {'Authorization': f'Bearer {sys.argv[3]}', 'Accept': 'application/vnd.github+json'}
try:
    req = urllib.request.Request(url, headers=hdr)
    sha = json.loads(urllib.request.urlopen(req).read())['sha']
except urllib.error.HTTPError:
    pass
body = {'message': msg, 'content': b64, 'branch': 'main'}
if sha: body['sha'] = sha
req = urllib.request.Request(url, method='PUT',
    headers={**hdr, 'Content-Type':'application/json'},
    data=json.dumps(body).encode())
try:
    d = json.loads(urllib.request.urlopen(req).read())
    print(target, '->', d.get('commit',{}).get('sha','?')[:7])
except urllib.error.HTTPError as e:
    print(target, 'FAIL', e.code, e.read().decode()[:200])
```

### Step 3 — for each pending file

For binary (pptx/docx) files, use the Drive MCP `download_file_content` tool which returns base64 in `content[0].embeddedResource.contents.blob`. Write that blob string to a `.b64` file (no decode needed), then:

```bash
cat /tmp/file.b64 | python3 /tmp/kbt_push.py "templates/X.pptx" "import: X" "$TOKEN"
```

For markdown (text), `read_file_content` returns text — base64-encode it before piping:

```bash
python3 -c "import base64,sys; sys.stdout.write(base64.b64encode(open('/tmp/file.md','rb').read()).decode())" \
  | python3 /tmp/kbt_push.py "docs/archive/X.md" "import: X" "$TOKEN"
```
