## Session-summary

Six gaps from v6 are now closed:

1. **`kbt_teams` RLS turned on** with anon SELECT/INSERT/UPDATE policies (DELETE intentionally not granted to anon — host-side only).
2. **Two-captain constraint** — partial unique index `kbt_team_member(team_id) WHERE is_captain=true`. The DB now refuses to mark two members as captain on the same team.
3. **`kbt_live_answers` extended** with `round`, `question_number`, `is_correct`, `points_awarded` columns — ready for future per-question answer-submission UI (host UI doesn't write to it yet; uses `trial_scores` for correctness).
4. **Per-question correctness tracking** — host live-quiz UI now shows a ✓ Right / ✗ Wrong panel per team on every question (alongside the Gambler wager panel for Gambler questions). Marks persist to `trial_scores` with `question_number > 0`. The round total at `question_number = 0` is auto-recomputed as the sum of per-question rows for that team+round, so existing flows stay consistent.
5. **Parameterised rounds** — score table, teamScores init, average calc, and renderScoreTable all derived from `_activeRounds` (loaded dynamically from `quizItems`). No more hardcoded R1/R2/R3.
6. **Tonight's Schedule from `kbt_quiz`** — replaces hardcoded "Round 1 General Knowledge / Round 2 Theme Round / Round 3 Final Round". Now derives one row per round from the loaded quiz items, with the dominant question category as the round label and "(N qs)" suffix.
7. **Real KBT logo on Slides title slide** — mirrored `https://www.knowbrainertrivia.com.au/img/logo/kb_logo_white.png` to `/assets/kb_logo_white.png` in the repo, and slides-export.js now uses `createImage` with that URL on the title slide instead of the text "KNOW BRAINER" lockup.

Plus: **wrap.html got a new "Questions answered correctly" slide** that surfaces N-out-of-M plus a percentage and a tone-appropriate caption (≥80% gold "smashed it", ≥50% green "solid effort", <50% magenta "rough night"). Only renders when per-question rows exist for the team.

End-to-end smoke verified against synthetic data: 4-round event with 5 questions/round and 11/20 correct → wrap renders 10 slides, accuracy slide shows "11/20 · 55% — solid effort". Cleanup confirmed (0 ZZ_ rows).

`trial_scores` is now used for two purposes:
- `question_number = 0` rows → canonical round total per (event_code, team_name, round)
- `question_number > 0` rows → per-question correctness records, points = 1 (or `quiz_points`) if right else 0

Both can coexist — the host UI keeps `question_number = 0` in sync with `SUM(question_number > 0)` whenever a per-question mark is changed.

## How per-question correctness works

**On the live quiz page,** below every non-Gambler question, a blue "Mark Q{n} (Round {r}) — N pt(s)" panel appears with one row per team:
- ✓ Right button (turns green when active) — writes `points = q.points`
- ✗ Wrong button (turns red when active) — writes `points = 0`
- Persisted to `trial_scores` with `question_number = q.num`
- Round total at `question_number = 0` is rewritten in the same call as the sum of all question_number > 0 rows for that team+round

**On the host's score table** (Update Scores page), each team row now has dynamic R{r} input cells, one per round in `_activeRounds`. The host can still type round totals directly — that writes `question_number = 0`.

**On revisit** (host navigates back to a previously-marked question), the panel hydrates from `_perQuestionRowCache` (loaded from `getAllScores` at launchEvent time) and shows the existing marks.

**On the wrap page,** if any per-question rows exist for the team, the new "Questions answered correctly" slide appears between the hero and the rank slide, showing N/M and the percentage. If no per-question data, that slide is skipped silently and only round totals are used.

## How parameterised rounds work

`_activeRounds` is a JS array of round numbers, populated from `quizItems` after launchEvent:
```js
_activeRounds = [...new Set(questions.map(q => q.round))].filter(Number).sort((a,b)=>a-b);
```

Default fallback `[1, 2, 3]` is used pre-launch. Helpers:
- `emptyRoundScores()` returns an object keyed `{R1: 0, R2: 0, ...}` per active round
- `totalForTeam(teamId)` returns the sum across all active rounds
- `renderScoreTableHeader()` rewrites the `<thead>` to one `<th>R{n}</th>` per active round

A 5-round event will now render 5 input columns. A 2-round event renders 2.

---

## How Tonight's Schedule renders

`renderSchedule()` is called from `launchEvent` after questions load. It groups `questions` by round, picks the most common question type per round as the label, and emits one `<tr>` per round with:
- Time = 7:00 PM + (round_index × 30 min)
- Round = "Round {n}"
- Type = "{TopType} ({count} qs)"

If quizItems is empty, shows a single "No quiz items loaded yet" row.

## Outstanding gaps after this session

Smaller list than before. None are blockers for a real event:

- **`kbt_live_answers` host UI not wired** — table extended, but the host UI doesn't push answers there. Currently per-question correctness lives in `trial_scores`. If you want player-submitted answer text (vs host-marked correctness), that's the next layer.
- **No PDF export** — the slides export is good but downloading as PDF still requires a manual export from Slides.
- **VVDS Fifties font** — the actual title text in slides still uses Open Sans (Slides API can't embed custom WOFF2). The logo image has the font baked in, so the title slide does look right; sub-headings do not.
- **Score reconciliation edge case** — if the host uses both per-question marks AND types in a manual round total, the manual total gets overwritten next time a per-question mark is clicked for that team+round. Accept by design (per-q marks are authoritative when used).
